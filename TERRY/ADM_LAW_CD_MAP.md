## 1. 법정동 코드 (10자리)

```
1 1 1 4 0 1 0 1 0 0
│ │ └─┘ └─┘ └───┘
│ │  구  읍면동 리
│ 시도
전국
```

| 자리 | 의미 | 예시 |
|------|------|------|
| 1~2 | 시도 | `11` = 서울 |
| 3~5 | 시군구 | `140` = 중구 |
| 6~8 | 읍면동 | `101` = 소공동 |
| 9~10 | 리 (읍면 하위) | `00` = 동 단위 |

**예시:**
```
1114010100 = 서울(11) 중구(140) 소공동(101) 00
1114010600 = 서울(11) 중구(140) 명동2가(106) 00
```

**특징:**
- 법적 주소(지번주소) 기준
- 거의 바뀌지 않음
- 실거래가 API, 등기 등에서 사용
- SOHOBI: `LAW_DONG_SEOUL.LAW_CD`

## 2. 행정동 코드 (8자리)

```
1 1 1 4 0 5 1 5
│ │ └─┘ └───┘
│ │  구  행정동
시도
```

| 자리 | 의미 | 예시 |
|------|------|------|
| 1~2 | 시도 | `11` = 서울 |
| 3~5 | 시군구 | `140` = 중구 |
| 6~8 | 행정동 | `515` = 명동 |

**예시:**
```
11140515 = 서울(11) 중구(140) 명동(515)
11110630 = 서울(11) 종로구(110) 청운효자동(630)
```

**특징:**
- 실제 행정 업무(주민센터) 단위
- 분동/합동으로 코드가 바뀔 수 있음
- 상권분석(SANGKWON_SALES), 지도 클릭에 사용
- SOHOBI: `adm_cd` (8자리)

## 3. 법정동 앞 8자리 (emd_cd)

```
11140101  ← law_cd(10자리) 앞 8자리
```

**예시:**
```
법정동코드: 1114010100
emd_cd:    11140101  (앞 8자리, 리 코드 제거)
```

**왜 쓰냐?**  
서울 열린데이터광장 실거래가 API가 이 단위로 필터링하기 때문

**SOHOBI 변환 흐름:**
```
adm_cd(행정동 8자리)
  → LAW_ADM_MAP
  → law_cd(법정동 10자리)
  → emd_cd(앞 8자리)
  → API 조회
```

## 4. 센서스행정동 코드 (10자리)

```
1 1 1 4 0 5 1 5 0 0
│ │ └─┘ └───┘ └─┘
│ │  구  행정동  세분
시도
```

**특징:**
- 통계청 인구주택총조사 기준 (5년마다 개편)
- 행정동을 더 세분화 (대형 동을 2~3개로 쪼갬)
- 생활인구 데이터에서 사용
- 행정동코드(8자리) + 순번(2자리) 구조

**예시:**
```
행정동: 11140515 (명동)
센서스: 1114051500 (명동 전체)
        1114051501 (명동 세분 1)
```

## 5. 250M 격자 코드

서울시 250M 격자 생활인구 CSV 기준

### 격자 ID 구조

실제 CSV 컬럼명: `GRID_ID` 또는 `GRD_ID`

```
형식 예시: 5174085750
           └──┘└──┘└──┘
           좌표X  좌표Y
```

또는 TM 좌표 기반:
```
형식 예시: G0055B0012
           G + X인덱스(4자리) + B + Y인덱스(4자리)

X좌표(미터) = X인덱스 × 250
Y좌표(미터) = Y인덱스 × 250
격자중심    = (X+125, Y+125)  ← 이 좌표로 히트맵 찍음
```

### 좌표계 변환

서울시 250M 격자는 **GRS80 TM 중부원점 (EPSG:5181)** 기준:

```python
from pyproj import Transformer

transformer = Transformer.from_crs("EPSG:5181", "EPSG:4326")

# 격자 중심 좌표 계산
x_idx = int(grid_id[1:5])   # G 다음 4자리
y_idx = int(grid_id[6:10])  # B 다음 4자리

x_tm = x_idx * 250 + 125    # TM X (미터)
y_tm = y_idx * 250 + 125    # TM Y (미터)

lat, lng = transformer.transform(x_tm, y_tm)
```

### 주요 컬럼 (추정)

| 컬럼 | 설명 |
|------|------|
| `GRID_ID` | 격자 ID |
| `STD_YM` | 기준 연월 (YYYYMM) |
| `TIME_SLOT` | 시간대 (0~23) |
| `POPLTN_CNT` | 생활인구 수 (`*` = 3명 이하 비식별) |
| `MALE_POPLTN` | 남성 인구 |
| `FEMALE_POPLTN` | 여성 인구 |

> CSV 실제 컬럼은 파일 다운로드 후 확인 필요

### 행정동과 연결

```
250M 격자 ID
  → 격자 중심 좌표 (위경도)
  → PostGIS ST_Contains 또는 좌표 포함 여부
  → 행정동 폴리곤(seoul_adm_dong.geojson)
  → adm_cd 매핑
```

SOHOBI에서는 GeoJSON 경계와 격자 좌표를 비교해서 행정동 단위로 집계 가능
