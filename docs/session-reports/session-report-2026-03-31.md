# 세션 리포트 — 2026-03-31

## 작업 개요

행정 에이전트의 LLM 일반 지식 의존성 해소를 위한 **Knowledge Base 플러그인** 구현 및 로컬 검증 완료.

---

## 배경 및 문제점

기존 `admin_agent.py`는 행정 절차(영업신고, 보건증 등) 질문에 대해 GPT-4o의 학습 데이터에만 의존하여 응답하고 있었음.

- 법령 조항 오인용 가능성 (예: 소방 법령 혼동)
- 서식 번호·관할기관 오류 위험
- Sign-off A1~A5 기준 충족 불안정

---

## 구현 내용

### 신규 파일

#### `integrated_PARK/data/admin_procedures.json`

법령 검증된 5개 행정 절차 데이터를 구조화한 Knowledge Base.
`CHOI/vectorSearch/refined_law_data_1.json`(legal-index 원본 데이터)과 교차 검증하여 모든 법령 인용을 확인함.

| 절차 ID                 | 절차명                 | 근거 법령                                         |
| ----------------------- | ---------------------- | ------------------------------------------------- |
| `food_biz_report`       | 일반음식점 영업신고    | 식품위생법 제37조, 시행규칙 제42조                |
| `hygiene_education`     | 영업자 위생교육        | 식품위생법 제41조, 시행규칙 제52조                |
| `business_registration` | 사업자등록             | 부가가치세법 제8조                                |
| `health_certificate`    | 건강진단결과서(보건증) | 식품위생법 제40조                                 |
| `fire_safety`           | 안전시설등 완비증명서  | 다중이용업소의 안전관리에 관한 특별법 제9조 제5항 |

> **법령 검증 메모**: 기존 계획안에서 소방 법령을 "소방시설 설치 및 관리에 관한 법률 제22조"로 기재했으나, 해당 조항은 자체점검 규정임을 확인하고 올바른 조항으로 수정함.

#### `integrated_PARK/plugins/admin_procedure_plugin.py`

Semantic Kernel `@kernel_function`으로 등록된 KB 조회 플러그인.

- 키워드 매칭으로 절차 ID 식별 (영업신고, 보건증, 소방 등 13개 키워드)
- KB 조회 성공 시: 법령·서식·관할기관·절차·서류 목록 반환
- KB 미등재 질문 시: `"찾지 못했습니다"` 반환 → LLM 일반 지식 폴백
